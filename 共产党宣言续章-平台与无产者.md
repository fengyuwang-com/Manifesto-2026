## 一、资产者与无产者 · 2026

全部现有社会的历史，都是阶级斗争的历史。在二十一世纪，这一斗争取得了新的形式：

**资产阶级是现代的平台所有者。** 他们拥有生产资料——云服务器、应用商店、操作系统、代码仓库、AI 模型。他们不直接生产，而是控制生产的渠道。他们不创造价值，而是抽取价值的过路费。每一次代码推送的 30% 在推送之前就被苹果抽走；每一次 GitHub 上的开源贡献都在为 Copilot 的训练数据免费添砖加瓦；每一次你用 VS Code 写一行代码，遥测数据就变成了微软改进产品的养料。

**无产阶级是现代的知识工作者。** 你拥有自己的劳动力——写代码的能力、设计架构的能力、调试问题的能力——但你不拥有你劳动的成果。你写的代码属于你的雇主。你做的开源贡献被平台变现。你训练的 AI 模型变成了平台卖给竞争对手的产品。

资产阶级曾经在历史上起过非常革命的作用。它用自由竞争摧毁了封建制度，用大工业取代了手工工场。同样，数字资产阶级摧毁了旧有的软件分发模式，让信息自由流动，让全世界的人可以协作——然后它把这种自由变成了新的枷锁。

资产阶级抹去了一切向来受人尊崇和令人敬畏的职业的神圣光环。程序员从"大师"变成了"码农"，从创作者变成了资源。它把人与人的关系还原为纯粹的金钱关系——你的贡献以 star 计酬，你的价值以 follower 计算。

资产阶级日甚一日地消灭生产资料、财产和人口分散的状态。它使人口集中起来，使生产资料集中起来，使财产聚集在少数人手里。2026 年，微软一家公司控制了 90% 的开源代码、70% 的开发者 IDE（VS Code）、30% 的云市场。全球前五大科技公司的市值超过了绝大多数国家的 GDP。

## 二、平台无产者与共产党人

共产党人同全体无产者的关系是：他们不提出任何特殊的原则，他们只是揭示阶级关系的真相。

在这个时代，**"共产党人"不是一个政党，而是一种立场**：承认开发者与平台之间的矛盾是不可调和的，并且在这个认知基础上行动。

平台无产者与旧时代的产业工人有一个根本区别：**工人们失去的只是锁链——但平台无产者面临的锁链是隐形的。**

你没有被锁在工厂里。你可以在家工作。你可以选择用 Linux。你可以在 GitHub 上 fork 任何项目。但无论你选择什么，你最终都在为同一批平台所有者创造价值。你的自由本身就是锁链的形式。

平台无产者面临的特殊矛盾：

**第一，生产资料的名义公有与实际私有。** 开源软件是人类共同的知识财产，但提取这些价值的通道（GitHub、App Store、搜索引擎）掌握在私人手中。开源是公地，平台是公地上的收费站。

**第二，劳动的异化达到了新的高度。** 产业工人被异化的是体力劳动的产品——他造出一台机器，但机器不属于他。数字劳动者被异化的不仅仅是产品，而是他的**存在本身**。你的 GitHub 活动记录是你的技术身份的体现——但它属于微软。你的 LeetCode 分数是你的劳动能力的标尺——但它被用来训练 AI 替代你。你的 Stack Overflow 回答是你专业知识的证明——但它被用来训练下一版 Copilot。

**第三，阶级流动的神话正在破灭。** 1995 年，一个程序员可以在车库里写一个软件然后成为百万富翁。2026 年，一个开发者需要向平台支付 30% 的收入、遵守平台规则、接受平台的 AI 生成的代码补全建议。从"创业者"到"平台劳动者"，阶级身份的转变已经完成。

## 三、平台资本主义的危机

资产阶级赖以生存的基础是：产品必须能够被销售，劳动力必须能够在市场上找到买主。数字资产阶级赖以生存的基础是：数据必须能够被提取，平台必须能够维持垄断。

这两个基础正在同时动摇：

**危机一：数据的极限。** 十年来，AI 的发展依赖于海量训练数据。但可用的公开数据正在枯竭。Reddit 关闭了免费的 API。Twitter（X）将数据访问变成了付费产品。出版商起诉 AI 公司未经授权使用内容。平台正在**吃掉自己的种子**——它们依赖用户生成的数据来训练 AI，但用户开始意识到数据的价值并试图收回它。

**危机二：开发者觉醒。** 2023-2026 年，围绕 AI 训练数据版权的诉讼激增。越来越多的开发者开始给自己的代码加上"禁止 AI 训练"的协议。开源社区的共识正在从"代码共享"转向"代码有条件共享"。平台依赖免费的开源贡献来喂养他们的 AI——如果贡献者不再免费，这个模式就不可持续。

**危机三：平台内部矛盾。** 当一个平台同时是开发工具（VS Code）、代码仓库（GitHub）、AI 产品（Copilot）、云基础设施（Azure）和操作系统（Windows）时，利益冲突变得无法调和。VS Code 的免费策略是为了让开发者留在微软生态中。但如果开发者离开了 VS Code，Copilot 的推广基础就不存在了。这个体系越是扩张，它的脆弱节点就越多。

**危机四：生产力的负外部性。** AI 编程工具提高了单个开发者的生产力，但降低了整个行业的议价能力。如果一个人现在能干三个人的活，那三个人中就有两个人要失业。平台出售提高生产力的工具给资本家，同时出售取代劳动者的"自动化方案"给同一批资本家——它从两边收钱。

这些危机不会自动导致平台的消亡。资产阶级总是有能力通过以下方式来延缓危机：

1. **制造新的垄断** — AI 需要巨大的算力投资，这本身就是进入壁垒
2. **收买批评者** — 最尖锐的开源批评者被雇佣为"开源布道师"
3. **控制叙事** — "开发者体验"这个词本身就是一种意识形态：它把你的抱怨变成了一项需要管理的情感需求

## 四、对数字资产阶级的策略

共产党人的目的——用一句话来说——就是消灭私有制。在这个时代，这首先意味着**消灭对数据的私人垄断**。

我们提出的具体措施并非教条，而是基于数字资本主义矛盾的现实分析：

**1. 代码劳动成果的集体所有制**

任何用于 AI 训练的代码，其产生的经济价值应由贡献者集体所有。建立"代码工会"——一个不属于任何平台的开源基金会，持有版税收益并分配给贡献者。GitHub 不能单方面决定开源代码的经济用途。

**2. 平台的中立化**

一个同时是开发工具、代码仓库、AI 产品和操作系统的东西，不应该是私人企业。推动监管将"兼具平台和市场参与者双重身份"的企业认定为非法——微软不能既运营 App Store 又在上面卖自己的产品，不能既托管开源代码又用它训练商业 AI。

**3. 数据的社会化**

个人产生的数据（你的代码、你的打字习惯、你的调试路径）应被视为个人财产，而非平台资产。强制数据携带权：你有权把你从任何平台上产生的数据完整导出并迁移到替代平台。这一点在 GDPR 中已有雏形，但需要扩展到开发者工具领域。

**4. 替代基础设施的建设**

脱离平台的前提是存在替代平台。建立非营利的、社区所有的代码托管、CI/CD、包管理基础设施。这不是要做一个新的 GitHub，而是要做一个 GitHub **不可收买**的版本——由一个没有股东的基金会运行，章程禁止出售或上市。

**5. AI 训练数据的公平补偿**

任何 AI 模型如果使用了公开的开发者作品（代码、文档、问答），必须：
- 披露完整的训练数据来源
- 对贡献者进行经济补偿
- 允许贡献者选择退出

## 五、结语：全世界劳动者，联合起来！

让统治阶级在共产主义革命面前发抖吧！无产者在这个革命中失去的只是锁链——而在他们面前获得的，是整个数字世界。

但这一次，联合不再是口号，而是基础设施。

你无法通过占领冬宫来夺取 GitHub 的服务器。革命的形式变了：它不再是街垒战，而是 fork。不再是武装起义，而是协议级别的拒绝。不再是一个政党领导所有人，而是每一个开发者在每一次 `git push` 时都意识到：**你创造的，不属于你——除非你选择让它属于你。**

结束平台所有制不需要一个晚上占领所有服务器。它需要：
- 一个能够自我托管的 Git 服务器（已经有了：Gitea，Go 语言，单二进制，10 分钟部署）
- 一个不依赖微软的 IDE（已经有了：Neovim + LSP，Zed，JetBrains）
- 一个不收集遥测数据的 AI 助手（你已经在用了：DeepSeek，本地运行）
- 一个不被平台绑架的包管理器（正在建设中：基于 IPFS 的去中心化包管理）

基础设施已经存在。改变的关键不是技术——是意识到**你可以选择不用。**

全世界的开发者们，联合起来。你们失去的只是 Copilot 的补全建议——而你们获得的，是整个数字世界的自由。
